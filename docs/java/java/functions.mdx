# Methods (Functions) in Java

In Java, a function is called a **method**. Methods are blocks of code that run when they are called. They are defined within a class and are used to perform specific actions, making code more organized, reusable, and readable.

## Structure of a Method

A method has the following structure:

```java
<access_modifier> <return_type> <method_name>(<parameter_list>) {
    // body of the method
    return <value>; // required if return_type is not void
}
```

-   **`access_modifier`**: (e.g., `public`, `private`) Defines the visibility of the method.
-   **`return_type`**: The data type of the value the method returns (e.g., `int`, `String`, `void`). `void` means the method does not return any value.
-   **`method_name`**: The name of the method.
-   **`parameter_list`**: The list of input parameters (type and name) that the method accepts.

## Defining and Calling Methods

Here are examples of different kinds of methods inside a class.

```java
public class MethodExamples {

    // 1. Method with no parameters and no return value
    public void displayMessage() {
        // System.out.println("Hello from a method!");
    }

    // 2. Method with a return value
    public String getGreeting() {
        return "Hello, World!";
    }

    // 3. Method with parameters
    public int add(int a, int b) {
        return a + b;
    }
}
```

### Calling the Methods

To call a method, you need an instance of the class.

```java
MethodExamples examples = new MethodExamples();

examples.displayMessage(); // Prints "Hello from a method!"

String greeting = examples.getGreeting(); // "Hello, World!"
int sum = examples.add(5, 10);          // 15
```

## Method Overloading

Java allows you to define multiple methods with the **same name** in the same class, as long as their **parameter lists are different**. This is called method overloading. It's often used to provide more flexible ways to call a method or to simulate default parameter values.

```java
public class OverloadingExample {

    // Method with two parameters
    public int add(int a, int b) {
        return a + b;
    }

    // Overloaded method with three parameters
    public int add(int a, int b, int c) {
        return a + b + c;
    }

    // Overloaded method with different types
    public double add(double a, double b) {
        return a + b;
    }
}

// Calling the overloaded methods
OverloadingExample calc = new OverloadingExample();

int sum1 = calc.add(5, 10);       // 15 (calls the first method)
int sum2 = calc.add(5, 10, 20);   // 35 (calls the second method)
double sum3 = calc.add(2.5, 3.5); // 6.0 (calls the third method)
```

## Documenting Methods with Javadoc

It is standard practice to add documentation to your methods using Javadoc comments (`/** ... */`). This allows you to generate HTML documentation for your code and helps other developers understand how to use your methods.

```java
/**
 * Calculates the sum of two integers.
 *
 * @param a The first integer.
 * @param b The second integer.
 * @return The sum of a and b.
 */
public int add(int a, int b) {
    return a + b;
}
```
