# Classes

Dart is an object-oriented language, and as such, it uses classes to define objects. A class is a blueprint for creating objects, providing initial values for state (member variables or properties), and implementations of behavior (member functions or methods).

## Basic Class

Here's a simple class with properties, a constructor, and a method.

```dart
class Dog {
  String name;
  int age;

  // Constructor
  Dog(this.name, this.age);

  // Method
  void bark() {
    print('Woof!');
  }
}

void main() {
  var myDog = Dog('Rex', 3);
  print(myDog.name); // Rex
  myDog.bark(); // Woof!
}
```

## Constructors and Initialization

Constructors are special methods used to create, or instantiate, an object of a class. The most common form is the generative constructor, which creates a new instance of the class.

### Initializer Lists

You can use an initializer list to initialize instance variables before the constructor body runs. This is useful for final variables.

```dart
class Point {
  final double x;
  final double y;
  final double distanceFromOrigin;

  Point(double x, double y)
      : x = x,
        y = y,
        distanceFromOrigin = x * x + y * y;
}

void main() {
  var p = Point(2, 3);
  print(p.distanceFromOrigin); // 13
}
```

## Simulating an Init Method

Dart doesn't have a built-in `init()` method that is automatically called upon object creation, unlike some other languages. Initialization is handled by the constructor. However, you can simulate this pattern in a few ways.

### Calling a Private `_init` Method

You can create a private method (e.g., `_init()`) and call it from the constructor.

```dart
class MyClass {
  MyClass() {
    _init();
  }

  void _init() {
    print('Initializing...');
  }
}

void main() {
  var instance = MyClass(); // Initializing...
}
```

### Using a Factory Constructor

A factory constructor can also be used to control the instance creation process and perform some initial setup.

```dart
class FileManager {
  final String path;

  // Private constructor
  FileManager._(this.path);

  // Factory constructor
  factory FileManager(String path) {
    print('Factory is setting up...');
    var fileManager = FileManager._(path);
    fileManager._initialize();
    return fileManager;
  }

  void _initialize() {
    print('File manager initialized for $path');
  }
}

void main() {
  var fm = FileManager('./my_files');
  // Factory is setting up...
  // File manager initialized for ./my_files
}
```

## Multiple Constructors

Dart allows you to have multiple constructors in a class. You can create a named constructor by using the `ClassName.constructorName()` syntax.

```dart
class Car {
  String make;
  String model;
  int year;

  // Main constructor
  Car(this.make, this.model, this.year);

  // Named constructor for a new car
  Car.newCar(this.make, this.model) : year = DateTime.now().year;

  // Named constructor from a map
  Car.fromMap(Map<String, dynamic> map)
      : make = map['make'],
        model = map['model'],
        year = map['year'];
}

void main() {
  var myCar = Car('Toyota', 'Camry', 2022);
  var newCar = Car.newCar('Honda', 'Civic');
  var carFromMap = Car.fromMap({'make': 'Ford', 'model': 'Mustang', 'year': 2021});

  print(myCar.make); // Toyota
  print(newCar.year); // 2025
  print(carFromMap.model); // Mustang
}
```

## Inheritance

Dart supports single inheritance. You can use the `extends` keyword to create a subclass.

```dart
class Animal {
  void makeSound() {
    print('Some generic animal sound');
  }
}

class Cat extends Animal {
  @override
  void makeSound() {
    print('Meow!');
  }
}

void main() {
  var cat = Cat();
  cat.makeSound(); // Meow!
}
```

## Mixins

Mixins are a way of reusing a class's code in multiple class hierarchies. You can use the `with` keyword to apply a mixin to a class.

```dart
mixin Flyer {
  void fly() {
    print('I can fly!');
  }
}

class Bird with Flyer {}

void main() {
  var bird = Bird();
  bird.fly(); // I can fly!
}
```

## Abstract Classes

Abstract classes cannot be instantiated. They are useful for defining interfaces. You can use the `abstract` keyword to declare an abstract class.

```dart
abstract class Shape {
  void draw(); // Abstract method
}

class Circle extends Shape {
  @override
  void draw() {
    print('Drawing a circle');
  }
}

void main() {
  var circle = Circle();
  circle.draw(); // Drawing a circle
}
```
